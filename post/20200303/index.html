自然言語処理の基礎

  • 形態素解析
  • 構文解析
  • 意味解析
  • テキスト分類
  • 機械翻訳
  • 情報抽出
    • 非構造化データであるテキストを構造化された表現に変換する
  • 質問応答
    • factoidの質問は短文で答えられる
    • non-factoidの質問はその逆

機械学習

  • 明示的なプログラムなしでコンピュータにデータから学習する能力を与える技術のこと
  • 機械学習パイプライン
    • 全体をデータの準備、モデル構築、デプロイというタスクに分けて構築
    • データを前処理、データから特徴量エンジニアリング、その後モデルの学習と評価
    • 特徴量エンジニアリングとは生データあるいは前処理済みのデータからモデルの学習に有用な情報を抽出すること

コーパス

  • 自然言語で記述あるいは口述され、コンピュータ上に格納されたデータのこと
  • クローリング、プログラムを使ってWEBから情報を取得
  • スクレイピング、クローリングした情報から特定の情報を取得
  • アノテーションツール、データにラベルをつけるツール
    • brat,docca

テキストの前処理

  • テキストのクリーニング
    • HTMLタグの削除など。Beautiful Soupとかlxmlつかうといいよ
  • 単語分割
    • MeCab, Juman++, Janome
  • 単語正規化
    • ねこ、ネコを同じ単語として処理できるようになる
  • ストップワードの除去
    • 一般的で役に立たない等の理由で処理対象外とする。「は」「の」「です」「ます」など。
  • 単語のID化
    • 単語をIDに置き換える

特徴量エンジニアリング

  • 機械学習アルゴリズムの性能が高くなるような特徴を生データから作成するプロセス
  • 特徴は「素性」と呼ばれることもある。
  • 質的変数
    • 名義尺度、順番に意味無し、カテゴリー、性別
      • そのまま整数値だと順番に意味がでちゃうので、one-hot-encodingで。
    • 順序尺度、順番に意味あり、、服のサイズ
      • 整数値に変換すると良い
  • 量的変数
    • 間隔尺度、順番に加えて値の間隔が等間隔であり、その差に意味がある、温度
    • 比例尺度、原点が定まっており、その比率に意味合がある。多重が50kgから1.2倍の60kgになった。
    • Binarizationと丸め処理をする。
  • テキストのベクトル化
    • 単語分割
    • ベクトル化
      • N-gramベクトル
        • n個連続させる。the cat, cat is, is out,など
      • シーケンスベクトル
  • 特徴量スケーリング
    • 特徴量の尺度を揃える処理
    • 正規化、特徴をある範囲に収まるようにスケーリング
    • 標準化、平均０，分散１にする

単語分散表現

  • 自然言語処理では単語を数値表現に変換して処理する
  • 単語の分散表現が使われることが多くなってきた。
  • 一つの単語を数百次元程度のベクトルで表す。
  • one-hotエンコーディングではベクトル間の演算で意味のある結果を得られない。単語が増えると次元を増やす必要あり。
  • 単語の分散表現は単語を低次元の実数値ベクトルで表す。
  • 基本的な考えは分布仮設。単語の意味はその近傍の単語によってわかる。
